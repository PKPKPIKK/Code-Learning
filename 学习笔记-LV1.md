# 10.选择语句/switch语句
    1.switch(表达式){
        case 值1:
            语句体1;
            break;
            case 值2:
                语句体2;
                break;
                ……
      }
    2.表达式的数据类型：byte、short、int、char、枚举、String
    3.case:被表达式匹配的值
    4.break:跳出当前循环，结束switch语句
    5.default:没有匹配的case，执行default中的语句
    注：case、default没有标准上下之分，位置可以任意书写，但是为了标准，
    case从小到大依次书写，default一般放在最后
        case穿透：若忘记书写break，此时就会触发case穿透
                  匹配上了，就会执行匹配的case中的语句，遇到break就结束整个switch语句
                  若没有break，就继续执行下一个case中的语句，直到遇到break或者结束switch语句
        switch新特性：
                    箭头标签：不会有case穿透现象
                    case后可以跟多个值
                    switch可以有运行结果，结合yield

# 22.static关键词
    1.静态成员变量：被当前类所有的对象共享（不属于对象，属于类；随着类的加载而加载，优先于对象存在）
        共享：赋值只要赋值一次；只要有一个对象修改了静态变量，其他对象再次访问的时候就是修改后的结果了
    2.调用方式
        方法一：类名调用（推荐）
        方法二：对象名调用
    3.内存解析：会在static加载时，在方法区生成一个静态区（JDK8以前），（JDK8以后）在堆内存生成一个静态区
    4.静态只能访问静态，非静态可以调用所有，静态方法无this
# 23.final关键词
    1.修饰变量： final int x;
        常量：只能赋值一次，不能更改；名字大写，多个单词用下划线隔开。
        要是基本数据类型，存储的数据值无法改变
        要是引用数据类型，那地址值无法改变，但是里面的内容可以改变
    2.不管是什么数据类型，用final修饰，那其记录的数据就不可更改
        修饰类的时候，就是最终类，不能被继承
        修饰方法，那就是最终的方法，不能被子类重写

# 25.继承
    1.继承：类与类的继承关系，关键字：extends
            子类继承父类，子类对象可以调用父类中的成员变量和方法
            把子类中重复的代码抽取到父类中（成员变量，成员方法都可以）
            子类应是父类的一种
    2.特点：只支持单继承，不支持多继承，单支持多层继承
    3.成员变量：就近原则，也会向上找到父类；super用父类当中的内容（最多访问上一层父类）;this是在本类的成员里去找
                可以被子类继承，哪怕是private私有，但是私有无法调用
    4.成员方法：在本类中调父类方法，super，或者直接用方法，默认前面有this;
                父类中的final、static不能被继承，可以被调用
                父类中的private不能被继承和调用
                父类中的虚方法可以被继承。(虚方法：不被final、static、private修饰)
    5.方法重写：父类中的方法不能满足子类的需求，就需要在子类中重写一遍，记得+@override
                父类中的代码一行都不想用，那就直接重写；要是还想用父类中的内容，可以用super直接调用父类方法，然后再添加新的
                名字和参数列表必须和父类保持一致
                访问权限必须大于等于父类（空着不写<proteced<public）
                返回值类型必须小于等于父类
    6.构造方法：父类里的构造方法不会被子类继承，需要super调用
                子类构造方法第一行默认有super()，不写JVM虚拟机也会给补上；要是想调用带参suer()，括号里必须写参数
                在创建对象时，先执行父类的构造方法，再执行子类的构造方法
                不能被子类继承，但是可以用super调用

# 31.接口（interface）
    1.子类中，无法抽取到父类，但是有多个子类拥有的方法，可以用接口定义统一的格式
    2.public interface 接口名{}
        接口和类之间用implenments
        子类要么重写接口中的方法，要么子类本身是一个抽象类
    3.其中的成员变量只能是常量，默认是 public static final
        其中没有构造方法
        在jdk7以前只能写抽象方法,默认是 public abstract
        jdk8，9就可以写有方法体的方法
    4.一个类继承多个接口，但是接口中有同名的方法，在类中只会显示一个，但是相当于重写了两个
    5.接口和接口之间可以单继承，也可以多继承，但是子类继承后要重写所有抽象方法
    6.拓展：
            JDK8后可以写有方法体的方法,需要＋default不可省略，否则默认为抽象方法
            两个接口中有一个同名的默认方法，那子类就必须强制重写此方法
            接口中的静态方法，必须+static,否则默认为抽象方法；只能用接口名调用
            JDK9有私有方法，private方法给默认的方法服务；private static方法给静态的方法服务
    7.适配器设计模式：设计模式就是各种规定的模板，套路
                     在接口中有很多方法，但是我只需要用其中一个方法，那就在中间加一个abstract类adapter，重写接口中的所有方法，空的方法体
# 32.内部类
    1.在一个类的里面，再定义一个类
    2.内部类可以直接访问外部类的成员，包括私有；
      外部类要访问内部类的成员，必须创建对象
      内部类是外部类的一部分，但是单独存在没有意义
    3.成员内部类：在JDK16之后，可以创建静态变量
                 若内部类没有用private修饰，则 Outer.Inner oi = new Outer().new Inner();
                 若内部类用private修饰吗，则在外部类用方法获取内部类；
                 内部类获取不同的变量（变量名重复）则用不同的修饰词：this，外部类.this（外部类对象的地址值）；
      静态内部类：只能访问外部类中静态变量和方法；想要访问非静态的需要创建外部类的对象,用对象去调用；
                 Outer.Inner oi = new Outer.Inner();
                 只要是静态的东西都可以用类名直接调用;
      局部内部类：将内部类定义在方法里，类似于局部内部类的局部变量
                 外界无法使用，需要在方法里创建对象使用
                 可以方位外部类的成员和方法内的局部变量
      匿名内部类：隐藏了名字的内部类,下面为实现关系、方法重写、创建对象
                 new 类名或者接口名（）{
                     重写方法；
                };
                其实是一个类的子类对象，或者是一个接口的的实现类对象；
                当方法的参数是类或者接口时，如果这个接口的实现类只使用一次，就不需要特别定义一个新的类，直接用匿名内部类简化代码；
